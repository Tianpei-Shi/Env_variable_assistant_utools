# 环境变量助手

一个现代化的环境变量管理 uTools 插件,采用 Modern SaaS 设计风格。

## ✨ 功能特性

- ✅ 环境变量组管理 - 创建、编辑、删除环境变量组
- ✅ 一键激活/停用 - 快速切换不同开发环境
- ✅ 系统变量查看 - 查看所有系统和用户级环境变量
- ✅ 值的隐藏/显示 - 保护敏感信息 (API Keys, Tokens)
- ✅ 一键复制 - 快速复制变量名或值
- ✅ 实时搜索 - 快速定位环境变量组
- ✅ 安全删除 - 删除前自动停用,防止环境变量残留

## 🚀 快速开始

### 在 uTools 中使用

1. 在 uTools 中输入: `环境变量`、`env` 或 `环境变量管理`
2. 点击 "创建环境变量组" 创建第一个环境变量组
3. 点击电源图标激活/停用环境变量组

### 创建环境变量组

```
组名称: Node.js Dev
描述: Node.js 开发环境

变量:
NODE_ENV = development
DEBUG = *
```

### 激活环境变量

点击环境变量组左侧的电源图标:
- 🟢 绿色 = 已激活
- ⚪ 灰色 = 未激活

## 📋 主要功能

### 1. 环境变量组管理

- **创建组**: 将相关环境变量组织在一起
- **编辑组**: 修改组名称、描述和变量
- **删除组**: 安全删除,自动清理环境变量

### 2. 值的隐藏/显示

- 点击 👁️ 图标切换显示/隐藏
- 保护 API Keys、Tokens 等敏感信息

### 3. 一键复制

- 📋 复制变量名
- 📋 复制变量值

### 4. 查看系统变量

- 点击 "查看系统变量" 查看所有环境变量
- 系统变量显示 🔒 图标 (只读)

## ⚙️ 技术栈

- React 19 + Vite 6
- Tailwind CSS 3
- Lucide React
- uTools Plugin API

## 💻 本地开发

### 安装依赖

```bash
npm install
```

### 启动开发服务器

```bash
npm run dev
```

访问: http://localhost:5173 (或自动选择的端口)

### 构建插件

```bash
npm run build
```

构建产物在 `dist/` 目录

### 在 uTools 中安装

1. 打开 uTools
2. 输入 `插件应用` → `开发者中心`
3. 点击 `安装开发插件`
4. 选择 `dist` 目录

## ⚠️ 注意事项

### 环境变量权限

| 类型 | 可修改 | 需要权限 |
|------|--------|---------|
| 用户级变量 | ✅ | 无需权限 |
| 系统级变量 | ✅ | 需要管理员权限 |

**Windows 用户**: 修改系统级环境变量需要以管理员身份运行 uTools

### 环境变量生效

- ✅ 新打开的命令行/终端立即生效
- ❌ 已打开的应用需要重启

### 安全建议

1. ⚠️ 谨慎修改 PATH 等系统关键变量
2. 🔒 使用 "隐藏值" 功能保护敏感信息
3. 💾 删除前确认,操作无法撤销

## 🔧 常见问题

### Q: 环境变量未生效?

**解决方案**:
1. 确认环境变量组已激活 (绿色图标)
2. 重启目标应用
3. 打开新的命令行窗口

### Q: 删除后环境变量残留?

**解决方案**:
插件已优化删除逻辑,删除前会自动停用并清理所有环境变量,不会出现残留。

### Q: 插件无法显示?

**解决方案**:
1. 更新 uTools 到最新版本
2. 重新安装插件
3. 查看开发者工具控制台错误 (F12)
